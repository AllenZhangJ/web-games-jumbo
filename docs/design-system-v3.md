# v3 视觉与动作系统

## 基线

v3 使用真实 Three.js 几何、正交相机和统一光照，视觉方向为“柔和浅灰影棚中的极简几何跳台”。它参考 `web-jump` 的立体动作语言，同时保留本项目的左右数值候选。

概念图：[`public/assets/concept/web-jump-three-v3.png`](../public/assets/concept/web-jump-three-v3.png)（1倍源尺寸 `941 × 1672`）。

![v3 竖屏三维视觉概念](../public/assets/concept/web-jump-three-v3.png)

概念图是构图、材质和层级基线，不是要在运行时显示的背景贴图，也不是对每个像素位置的硬编码规格。最终运行画面必须由 3D 几何、相机、光照、阴影和 WebGL HUD 动态生成。

## 颜色令牌

| 令牌 | 色值 | 用途 |
|---|---|---|
| `scene.background` | `#D8DDE2` | 世界清屏色和柔和影棚背景基色 |
| `character.primary` | `#E53935` | 角色头部、身体和主要跳跃载体 |
| `choice.accent` | `#16A6A1` | 当前选中的候选、积极引导和关键操作反馈 |
| `text.primary` | `#263238` | HUD 数值、运算符、按钮图标和高对比信息 |

上述四个色值是 v3 主题的权威色值。明暗面、阴影、禁用态和透明度应由材质、光照和 alpha 从主色派生，不要为每个状态重新引入高饱和品牌色。

### 中性材质

- 当前平台使用暖白/浅米白材质，靠光影而不是边框表达体积。
- 未选候选使用同系中性色；选中时可转为 `#16A6A1` 或添加青色强调。
- 历史平台降低明度、对比度或透明度，不使用新颜色抢占注意力。
- 阴影使用冷灰且边缘柔和；不烘焙进平台或角色贴图。

## 场景构图

### 正交视角

- 使用正交相机和稳定的斜向俯视，形成清楚顶面与侧面，不用广角透视制造夸张缩放。
- 相机保持世界方向一致，让玩家能稳定比较左右两条路线。
- 连续落地后平移 `worldRoot` 或 Camera Rig 完成构图推进，不突然重置角色屏幕位置。

### 可见世界层级

常态构图明确展示：

1. **一个当前平台**：位于视口下半部，承载红色角色。
2. **两个候选平台**：位于前方左右，运算符在上方或顶面保持高可读。
3. **一个历史平台**：在视口底部或后方退场区部分可见，用来表达连续行进。

世界模型可为逻辑和回收保留更多有界历史，但首要视觉层级必须围绕上述 `1 + 2 + 1` 结构，不让旧平台形成噪音。

## 几何与角色

- 平台以 Cube 和 Cylinder 为基础词汇，后续特殊平台也应保持低面数、大轮廓和可判断顶面。
- 角色为 `#E53935` 的基础几何 Rig：球形头部 + 锥柱/柱体身体，不继续使用狐狸精灵图。
- 基础几何的轮廓和形变就是主要动作反馈，不用 2D 帧动画覆盖。
- 平台真实碰撞尺寸由 Core 决定；表现压缩不改变碰撞顶面。

## HUD

HUD 只保留完成当前决策所必需的信息：

- 左上：当前值。
- 顶部居中：目标值。
- 右上：剩余步数。
- 需要时提供暂停、重开与结算覆盖层，但不扩展为浮夸的顶部仪表盘。
- 左右候选的运算符和结果就近显示，选中候选使用青色强调。
- 安全区底部放置两个独立悬浮按钮：`↖` 固定左路、`↗` 固定右路；按钮之外的世界区域不触发跳跃。

文字使用 `#263238`、粗细适中的无衬线字形，数字在动态变化时保持宽度稳定。HUD 必须在同一 WebGL Canvas 的独立 Scene 中渲染，并遵循平台安全区。

### 明确删除

- 无节拍轨、无节拍指针、无常驻连击信息。
- 无狐狸角色、无精灵图状态切换。
- 无数据城市、霓虹远景、透明城市前景或多层城市视差。
- 无贯穿整屏的常驻底部操作轨道；仅保留两个安全区悬浮箭头按钮，蓄力仍由按钮按压、角色和平台压缩共同反馈。
- 无大面积渐变卡片、多色发光描边或与几何影棚无关的装饰。

## 动作语言

- **蓄力**：身体变矮变宽，当前平台轻微下压，形变连续可读。
- **释放**：身体和平台快速回弹，进入 Core 轨迹。
- **空中**：角色局部完成一周前空翻，根节点始终与 Core 位置一致；轻量拖尾传达方向与速度。
- **落地**：短促压缩和回弹，辅以有界粒子和动态阴影。
- **世界推进**：落地后先停留约 `80 ms`，再让世界原点与相机焦点共享约 `440 ms` 的 `ease-in-out cubic` 位移；新候选的出现与这次推进重叠，而不是再次触发独立追赶。
- **失败**：根据 Core 已经给出的短跳、越过或边缘失败类型，播放坠落、倾倒或滑落；动画不修改失败结论。

动作可以有弹性，但总体要干净、短促和物理可读，不以长时间闪烁或屏幕特效替代身体反馈。

## 响应式与验收

- 竖屏是主设计方向，概念比例为 `941:1672`；不把源尺寸当作唯一运行分辨率。
- 长屏优先增加场景上下呼吸空间，不拉伸几何或放大 HUD。
- 较宽视口保持主场景居中且同时可读左右候选，不让平台随桌面宽度无限分离。
- HUD 必须落在安全区内；世界平台可以进入视觉边缘，但交互目标不得被刘海、圆角或系统手势区遮挡。
- 底部箭头按钮最小触控高度约 `56 CSS px`，必须锚定实际安全区底部，而不是固定设计画布的底边。
- 视觉验收至少覆盖待机、选中左/右、最大蓄力、空中最高点、偏心成功落地、短跳、越过、边缘失败、暂停、胜利和失败。

本机截图与概念图一致不能代替微信和抖音真机验收。
