# ADR-025：Arena S9.5 使用预注册、隐藏分组和逐 Tick 可复验的真人公平性研究

- 状态：已接受；S9.5a 合同、采集端口与复验 CLI 已实施，独立采集工作台和真实样本待完成
- 日期：2026-07-18

## 背景

S9.3 的 900 局配对 Bot benchmark 已证明三档相对能力、确定性和 11 条命候选，但脚本玩家不能代表真人。少量内部试玩也不能回答新玩家是否能在 10 秒内理解操作、三档是否形成合理胜率梯度、机器人行为是否自然，以及 2～3 分钟目标是否在真实触控下成立。

若看到结果后再改样本量、排除条件或胜率区间，会产生选择性停止和调参污染。若研究专门覆盖隐藏难度，采集路径又不等同于 Product QuickMatch，就无法证明结果属于实际首版。

## 决策

### 固定 V1 研究合同

`arena.stage9.human-fairness.v1` 在采集前固定：

- 仅招募已同意、没有 Arena 既往经验、没有参加过本研究的新玩家。
- Web、手机、竖屏、触控；每名参与者连续完成三局。
- easy、normal、hard 三个平行组，每组至少 30 名完成者，共至少 90 名完成者、270 局。
- 参与者提示只描述“1v1 对手”，不显示机器人、真人、难度、arm 或 seed，也不声称真人在线。
- 每个参与者只进入一个 arm；按连续 `enrollmentIndex` 做确定性 block shuffle。
- 研究不提供难度覆盖参数。每局 seed 必须经生产 `createMatchAssignment()` 天然选中该 arm，实际比赛仍走同一 QuickMatch、Bot、InputFrame、Rule/Core 和 Product 结果路径。

改变样本、环境、阈值、Prompt、候选 Balance 或 Bot Profile 必须新建 V2，不能覆盖 V1。

### 结论与门禁

主指标按“参与者 session”计算，避免把同一人的三局当作三个独立样本。三局中玩家胜局多于负局才计 session win，平局不计胜。

V1 预注册：

- easy 玩家 session 胜率 50%～80%，normal 30%～70%，hard 20%～50%。
- 总体 session 胜率 40%～60%，95% Wilson 区间宽度不超过 0.25。
- easy 与 hard 胜率差至少 0.15；相邻强度档允许的反向倒挂不超过 0.10。
- 完成率至少 85%，失效率不超过 10%。
- 7,200～10,800 tick 的对局占比至少 60%。
- 猜测对手是机器人的比例不超过 50%，公平性和自然度平均分至少 3/5，愿意再来一局至少 60%。
- 时长、感知和再来一局指标同时按 arm 与总体检查，防止总体平均掩盖某一难度异常。

样本不足时 Report 必须为 `incomplete`，即使早期指标已经异常；异常 gate 仍保留供观察。只有三个 arm 都达到预注册样本后，Report 才能给出 `ready` 或 `failed`，禁止根据早期结果选择性停止。

### 排除、隐私与完整性

排除原因在 Definition/Record 代码中预先枚举：未同意、既往 Arena 经验、重复参加、提示偏离、操作员协助、平台/形态/方向/输入不符，以及运行失败或协议失效。不得事后新增“看起来异常”的排除理由。

只保存去标识参与者 ID、操作员 ID、结构化资格字段、自评和 Replay；不在 Bundle 中保存姓名、手机号、设备唯一标识或聊天。`enrollmentIndex` 必须从 0 连续，退出和失效也必须形成终态 Record，不能静默删掉不利样本。

### 同一产品路径与证据复验

Product Match 增加默认关闭、同步、只读的完成端口。端口只收到深冻结的 `{ result, replay }`，不认识 Study，不参与命中、随机、结算或生命周期。Study Capture Session 注入预注册 seed 与该端口；缺失 Replay、重复分配、异步 sink 或身份不一致均 fail closed。

证据 Bundle 绑定唯一 40 位 commit、build ID、Definition hash 和内容寻址 Replay。Node CLI 会：

1. 防止路径逃逸、文件替换、硬链接复用、内容复用、大小或 SHA-256 不一致。
2. 以冻结 Product 配置严格重放每局。
3. 用生产 Bot Profile、行为 seed 和个性 seed 逐 Tick 重新生成 `player-2` 输入并逐帧比对。
4. 重建 Product Match Result，再计算 arm/总体 Report。

工程测试、模拟参与者或手写 JSON 都不能替代真实人的 Record。

## 后果

- S9.5 结论可绑定到准确候选并独立复验，机器人不能通过未来信息、特殊难度覆盖或另一条输入路径获得优势。
- 研究至少需要 90 名合格完成者，实际招募数会因退出和预注册排除高于 90。
- Replay 证据体积和复验时间增加；它们只存在于研究/CLI 路径，不进入生产权威热路径。
- S9.5a 工程基础完成不等于 S9.5 冻结。独立采集工作台、知情同意流程、真实参与者和最终 Bundle 仍是外部阻断项。

## 拒绝的替代方案

### 让参与者选择简单、普通、困难

这会暴露研究变量并偏离首版隐藏随机难度，拒绝。

### 为研究直接覆盖 Bot 难度

专用覆盖无法证明生产匹配路径，且可能掩盖分配错误。V1 只搜索天然产生目标隐藏难度的 production seed。

### 用脚本 Bot 或内部开发者代替真人

只能证明工程相对能力，不能证明新玩家理解、触控行为、自然度或感知公平性，拒绝。

### 告诉玩家“对手是真人”

首版只隐藏机器人身份，不伪造真人在线声明、社交信号或真实玩家资料，拒绝。
