# Arena V1 游戏规则

## 文档状态

已接受的第一版规则框架。具体平衡数值在无渲染模拟和实机测试中调整。

## 对局合同

| 项目 | Arena V1 规则 |
|---|---|
| 对局构成 | 1 名玩家 + 1 名隐藏本地对手，固定 1v1 |
| 对手实现 | 本地机器人；生产界面不暴露身份 |
| 机器人难度 | 简单、普通、困难，由 match seed 等概率随机；玩家不可选择、不可见 |
| 单局时长 | 最长 150 秒，目标体验 2～3 分钟 |
| 生命 | 每名角色 3 条命 |
| 传统生命值 | 无 |
| 淘汰 | 越过地图淘汰边界、跌入深渊或被规则区域淘汰 |
| 胜者 | 其他参赛者全部失去所有生命；超时按剩余生命和规则积分结算 |

重生时给予短暂无敌和可读的落点预告。具体时长属于平衡参数，不写死在产品文档中。

## 快速匹配合同

1. 玩家以已选择或随机的角色发起“快速匹配”。
2. 客户端创建 match seed，并分别从虚构对手形象池和三档难度池中抽取结果。
3. 难度默认按 `1:1:1` 等概率抽取；没有玩家可见的选择、标签或结果提示。
4. 客户端完成短暂的寻找对手和准备转场后，直接创建本地 `MatchSession`。
5. 生产 UI 可以显示中性的对手昵称、外观和准备状态，但不得显示“真人在线”、在线人数、聊天等没有真实后端来源的信息。

对手形象 RNG、机器人难度 RNG、地图 RNG 和装备 RNG 必须是从同一 match seed 派生的独立随机流。难度只记录在调试日志、批量统计和回放元数据中。

## 输入合同

每个参赛者无论由玩家还是机器人控制，每个逻辑 tick 只向 MatchCore 提交同一种输入：

```text
InputFrame
  tick
  moveX
  moveZ
  actionPressed
  actionHeld
```

机器人不能直接调用击飞、拾取、移动或淘汰函数。

## 动作键解析

动作键使用稳定优先级：

1. 淘汰、重生、硬直或明确不可行动时忽略输入。
2. 持有且可使用装备时释放装备技能。
3. 空中且允许下砸时执行下砸。
4. 位于明确可交互机关范围时触发机关。
5. 其他地面状态执行无装备基础推击。

当前将要执行的行为必须通过图标、轮廓或动作预备姿态表达。装备默认自动拾取，避免拾取和攻击争夺同一个动作键。

## 跳跃和回场

- 普通台阶、坡面和短距离地形连接可由移动系统自动通过。
- 自动跳跃不得越过明显危险或主动追踪移动目标。
- 被武器或机关击飞后，不自动把角色拉回安全平台。
- 空中下砸是动作键的主要空中语义。
- 主动跳跃和二段跳是否进入正式规则，需在首个地图验证后另行决定；角色表现系统可以预留对应动作语义。

## 角色模板

角色模板只允许有限差异：

- 移动速度。
- 体型和碰撞半径。
- 重量或击飞抗性。
- 转向与制动响应。

第一版不允许角色拥有专属主动技能。相同装备必须保持相同用途，角色参数只能改变使用节奏，不能改变装备类别。

## 装备合同

角色同时只持有一件主要装备。装备至少定义：

```text
EquipmentDefinition
  id / category
  range / targetShape
  windupTicks / activeTicks / recoveryTicks
  horizontalImpulse / verticalImpulse
  selfImpulse
  cooldownTicks
  defense / interrupt / dropPolicy
  presentationSemantic
```

首个垂直切片只实现：

- 大锤：短距离、明显前摇、高击退。
- 锁链：中距离命中、把目标拉向使用者。
- 盾牌：短时间防御，并能向面对方向冲撞。

装备掉落必须在地图中有高辨识度，并提前显示刷新点或倒计时。

局外解锁的新装备只会加入后续比赛的对称掉落池；玩家和机器人面对同一装备集合，任何角色都不能把已解锁装备直接带入比赛。

## 地图合同

地图以数据定义，而不是直接以 Three.js Mesh 作为玩法真相：

```text
MapDefinition
  bounds / killPlane
  spawnPoints / respawnPoints
  surfaces
  hazards
  pickupSpawners
  eventTimeline
  suddenDeathRule
```

首张地图包含：

- 深渊淘汰边界。
- 分阶段塌陷地面。
- 一个风场或旋转机关。
- 一个预告式装备刷新点。
- 对局后半段的缩圈或加速塌陷。

地图事件由逻辑 tick 和 seed 驱动。Renderer 只播放已经确定的预警和结果。

## 击飞规则

- 命中结果由装备、攻击方向、目标重量、防御状态和相对位置共同决定。
- 击飞先生成权威玩法事件，再由表现层播放硬直、拖尾、命中特效和镜头反馈。
- 不通过传统生命值清空淘汰角色。
- 第一版不引入不断累积的伤害百分比；如果固定击飞无法形成足够的对局节奏，再单独评审短时“失衡值”。

## 随机性

- 每局记录一个 match seed。
- 对手形象、机器人难度、地图事件、装备类别和机器人个性扰动使用独立随机流。
- 机器人难度不得改变地图和装备随机序列。
- 相同 seed 和相同 InputFrame 序列必须得到相同淘汰、拾取和胜负结果。

## 权威玩法事件

MatchCore 输出但不由 Renderer 创建的事件至少包括：

- `EquipmentSpawned`
- `EquipmentPickedUp`
- `ActionStarted`
- `HitResolved`
- `KnockbackApplied`
- `HazardWarningStarted`
- `HazardTriggered`
- `PlayerEliminated`
- `PlayerRespawned`
- `MatchEnded`
