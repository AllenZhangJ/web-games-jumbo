# 每批提交前检查清单

本清单适用于第 0 批、四个治理批次和最终终验。代码或文档写完不代表批次完成。

## 1. 范围和依赖

- [ ] 本批实际修改与路线图一致，没有偷跑后续产品功能。
- [ ] 依赖只沿允许方向流动，没有新循环依赖或平台泄漏。
- [ ] 没有同时保留两套长期玩法真相或不可删除的临时兼容层。
- [ ] 公共契约变更有 ADR、版本和调用方迁移说明。

## 2. 健壮性

- [ ] 所有外部输入、配置、平台返回值和存档都在边界验证。
- [ ] 非有限数、空值、错误 ID、越界数组、零尺寸和资源缺失有明确行为。
- [ ] 单个 Renderer、Feedback、Storage 或 cleanup 失败不会留下半提交状态。
- [ ] 所有资源和监听器有幂等销毁路径。

## 3. 竞态与异步

- [ ] 重复 start、restart、destroy 和并发启动有测试。
- [ ] 销毁后异步加载/回调不能恢复旧实例或写入状态。
- [ ] pointer/touch 归属、cancel、丢失 capture 和多指输入有测试。
- [ ] RAF、固定步长和时钟切换不会重复调度、追赶后台时间或产生负 delta。

## 4. 兜底和事务

- [ ] 候选生成、落地、存档、资源加载和平台 API 抛错有兜底。
- [ ] 失败不会让 Domain、World、Presentation 或存档处于互相矛盾状态。
- [ ] 兜底不会悄悄改变玩法结果。
- [ ] 不可恢复失败进入显式、可诊断的状态。

## 5. 边界与主流程

- [ ] 最短、理想、最长蓄力及短跳/成功/越过/边缘失败通过。
- [ ] 暂停、恢复、重开、胜利下一轮和失败重开通过。
- [ ] 当前平台、候选、历史和玩家位置在多轮后保持一致。
- [ ] 固定 seed 的整局主流程结果与基线一致，除非有已批准行为变更。

## 6. Web 生命周期和浏览器

- [ ] 页面加载非空，无框架错误覆盖层。
- [ ] resize、hide/show、pagehide/pageshow、pointercancel 行为明确。
- [ ] 长按不触发选择/菜单，左右按钮对应屏幕左右路径。
- [ ] 控制台没有未解释的 error/warn。
- [ ] WebGL context lost/restored 按本批承诺范围验证。

## 7. 测试与构建

- [ ] 相关最小测试通过。
- [ ] 全量测试通过。
- [ ] 本批新增分支、边界和失败路径有测试，而非只测试成功路径。
- [ ] Web、微信和抖音构建全部通过。
- [ ] 构建产物未泄漏 DOM、Node、错误平台 API 或开发资源。

## 8. 待提交审计

- [ ] `git diff --check` 通过。
- [ ] 暂存文件只属于本批范围。
- [ ] 没有 `node_modules`、`dist`、密钥、令牌、真实 AppID 或个人数据。
- [ ] 依赖锁、许可证和资源归属与实际变更一致。

## 9. 文档校准

- [ ] README 和文档索引仍能引导新成员。
- [ ] 仓库结构、模块职责和运行流程与当前代码一致。
- [ ] `governance/status.md` 明确已完成、未完成和不明确项。
- [ ] 测试数量、构建结果、包体和验证证据已更新。
- [ ] 没有把计划目标写成已完成功能。

## 10. 提交与推送

- [ ] 使用中文提交信息。
- [ ] 提交后工作区干净。
- [ ] 推送对应治理分支。
- [ ] 远端分支哈希与本地 HEAD 一致。
- [ ] 批次标签只在批次验收完成后创建。

任何一项不满足时，批次保持“进行中”，修复并重新验证。
