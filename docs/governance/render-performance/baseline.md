# 渲染性能基线

采样日期：2026-07-15

## 已验证事实

| 项目 | 当前基线 | 证据 |
|---|---:|---|
| 10 个内容菜单组合纹理 | 59,536,000 bytes | `performance-baseline.test.ts` 创建 10 个 `1220×1220×4` CanvasTexture。 |
| 原纹理缓存上限 | 96 entries | TextureManager 构造默认值；与纹理尺寸无关。 |
| 内容扩容后全量测试 | 24 文件、146 项 | 内容提交 `69b65da` 前完整测试。 |
| 独立会话 | 1,000 局通过 | `tests/session-soak.test.ts`。 |
| Renderer 资源 soak | 100 局平台资源有界 | `resource-soak.test.ts`；尚未覆盖 HUD 字节。 |
| 桌面诊断缓存峰值 | 约 81,016,448 bytes、40 textures | 内容全遍历诊断记录；由菜单组合缓存主导。 |

## 已知热路径

- 松开按钮后，状态由 charging 切到 jumping；HUD 可能在同一帧创建并上传顶部、状态、左右控制纹理。
- TailTrail 在采样点创建 `THREE.Vector3`。
- ParticleBurst 刷新实例时创建 `THREE.Euler`，即使没有活动粒子也会刷新全部实例矩阵。
- Replay action 当前同步写入平台 Storage；桌面写入很快，但移动浏览器实现仍会占用释放调用栈。

## 目标

| 指标 | 高档 | 低档 |
|---|---:|---:|
| 目标帧时间 | 16.67 ms | 33.33 ms |
| 松开到可见起跳 | ≤33 ms | ≤33 ms |
| 长任务 | 不得超过 50 ms | 不得超过 50 ms |
| UI 纹理预算 | ≤24 MiB | ≤10 MiB |
| 动态 UI 纹理预算 | ≤12 MiB | ≤6 MiB |
| 内容菜单 | 1 个可重绘纹理 | 1 个可重绘纹理 |
| 内容全遍历后的项目验收线 | ≤16 MiB | ≤10 MiB |
| 热路径 Three 对象分配 | 0 | 0 |
| 100 局后资源数量 | 回到有界稳态 | 回到有界稳态 |

真实 iPhone 帧时间、FPS、内存和长任务仍为待验证，不能用 Node 单测推断为达标。

## 第二批改进后的当前事实

- 内容菜单从每个组合一个 `1220×1220` 缓存纹理，改为一个 `1024×1024` 可重绘纹理。
- 390×844 浏览器遍历内容后的总纹理从诊断基线约 81 MB 降到 8,995,072 bytes。
- 菜单关闭后动态纹理立即释放，总纹理降到 6,332,928 bytes。
- 释放到起跳状态切换不再创建 CanvasTexture；真实 iPhone 帧时间仍待最终验收。
