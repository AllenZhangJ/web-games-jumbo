# 空中与落地卡顿修复验证

验证日期：2026-07-15

验证范围：本机 `http://127.0.0.1:4179/` 的 Web 生产构建；4173 被本机其他服务占用，因此最终构建使用独立端口避免旧资源缓存。按项目方决定，本轮不做手机真机调试。本文结果不能替代 iOS、Android、微信或抖音真机证据。

## 复现基线

在约 120 Hz 的本机浏览器中连续执行 3 次理想力度跳跃：

| 指标 | 修复前 |
|---|---:|
| 总帧 | 388 |
| 平均帧时间 | 9.42 ms |
| P95 / P99 | 17.3 / 25.4 ms |
| 最大帧时间 | 41.3 ms |
| 超过 25 ms | 4 帧 |
| 空中位置重复率 | 45.0% |

根因不是碰撞或数值计算：

1. 60 Hz 固定步长位置被直接显示在约 120 Hz 的 RAF 上，约一半显示帧重复同一位置。
2. 空翻使用起始角速度过高的 `easeOutCubic`，放大了固定步长量化。
3. 落地后的新平台标签、HUD 数值、粒子和镜头切换集中触发 GPU 纹理/材质工作。
4. 平台标签随预览值和选中态创建新 CanvasTexture，HUD 顶栏也随每个新数值持续增加缓存。

## 修复

- Application 在每个显示帧使用 `jump.elapsedMs + FixedStepClock.accumulator` 生成只读 `RenderMotionProjection`；碰撞、WorldState 和 GameState 仍只使用固定步长真相。
- 角色翻转改为 `easeInOutSine`，降低起跳瞬间角速度突变。
- 镜头/世界过渡取消 80 ms 静止延迟，使用 660 ms 连续正弦缓动。
- 平台标签改为按内置运算预建的固定 SpriteMaterial 池；纹理在 `load()` 期间上传，运行中只移动既有 Sprite，不再换图或重复使材质失效。
- 顶部 HUD 与状态条改为固定低分辨率 DynamicCanvasTexture，不再按每个数值/阶段增加缓存。
- 落地标签每帧最多处理一个；粒子、标签和 HUD GPU 上传按帧错开。
- World/HUD/粒子 Shader 在进入可交互状态前预热。
- 无 60 Hz tick 的高刷显示帧复用 State/World/Menu 快照；屏幕左右映射和帧指标使用预分配对象，降低 GC 压力。
- 高/低画质 DPR 上限为 1.75/1.5，阴影贴图为 512/256；两档继续保持 60 FPS 目标。
- 低画质不再以 30 FPS 为目标；高低两档均以 60 FPS 为目标，低档通过 DPR、阴影和特效容量降载。

## 自动化压力守卫

`npm run test:render-jank` 包含：

- 40 次、120 Hz 模拟跳跃；
- 2,000 个以上空中 RenderMotionProjection 帧；
- 空中重复投影率不高于 1%；
- 候选标签落地帧不创建新纹理，后续每帧最多处理一个；
- 标签复用既有 SpriteMaterial/Texture，不在落地时换图或重新编译；
- 粒子延迟到标签队列清空后的独立帧；
- 既有 100 轮 Renderer 资源 soak 与纹理预算。

## 证据边界

- 本机浏览器压力结果证明此次可复现的空中量化与落地 GPU 尖峰已消除，不承诺所有设备永不受系统调度、温控或后台任务影响。
- 手机、微信、抖音与低端设备的 10 分钟帧率、内存、温度和上下文恢复仍在发布清单中，不从本文推断通过。
