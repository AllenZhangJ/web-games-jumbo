# ADR-003：参考 `shenmaxg/web-jump` 并保留 MIT 归属

## 状态

已接受。

## 日期

2026-07-14

## 背景

[`shenmaxg/web-jump`](https://github.com/shenmaxg/web-jump) 是一个使用 Three.js 的蓄力跳台开源实现。它已经验证了本次重构希望获得的若干表现思路：

- 正交相机、环境光、方向光与实时阴影的 Stage。
- 用 Group 和基础几何组成的角色，以及蓄力压缩、回弹、空翻、拖尾、倾倒和滑落。
- Cube/Cylinder 平台、入场动画、世界整体平移与旧对象回收。

该仓库以 MIT 许可发布。但其代码使用较旧 Three.js API、Webpack/Babel、DOM 输入和单路线世界模型，不能无边界地整仓复制到本项目。来源仓库中的快递箱、魔方等纹理也不应在缺少独立资产来源证明时作为正式发行素材。

## 决策

1. 将参考基线固定为 commit `3fdcb17436f77ddb6664b9aad8f9c5fffdf0fe58`，后续评审可以重现当时参考的确切代码。
2. 可以复用或改编 Stage、角色分层、平台工厂、Tween 节奏、世界平移、拖尾、粒子与失败动作等实现思路或代码，但必须符合原 MIT 条款。
3. 所有从参考项目复制或实质改编的部分都保留 MIT 版权声明和许可全文。仓库保存 `licenses/web-jump-LICENSE`，发行归属汇总在根目录 `THIRD_PARTY_NOTICES.md`。
4. 不复制参考项目的 Webpack/Babel 工程、已过时 `Geometry` API、DOM 事件绑定、`Math.random()` 世界生成、单路线碰撞真相或用 Tween 决定落地的架构。
5. 用现代 `BufferGeometry`、当前 Three.js 模块、项目已有的确定性 RNG/Core 物理、平台契约和三端构建重写过时部分。
6. 视觉上可参考基础几何和动作语言，但正式发行素材应使用本项目可确认来源的原创或已授权资产。
7. 对 `web-jump` 的归属不表示原作者认可、赞助或与本项目存在商业关系。

## 与参考项目的边界

| 方面 | 可参考或改编 | 本项目必须保持或重写 |
|---|---|---|
| Stage | 正交相机、环境光和方向光、阴影构图 | 平台画布契约、单 Canvas HUD、resize 和安全区、上下文恢复 |
| Character | 几何分层、压缩和回弹、空翻、倾倒、拖尾 | Core 解析轨迹与碰撞真相；局部动画不影响结果 |
| Platforms | Cube/Cylinder、入场弹性、对象回收 | 稳定 ID、左右数值候选、真实尺寸与偏心落点 |
| World motion | 整个视觉世界平移的构图思路 | Core 绝对坐标不变，由 `worldRoot` 表现平移 |
| Build/runtime | 无 | Vite/esbuild、WebGL2、Web/微信/抖音适配与确定性测试 |
| Assets | 基础几何形状语言 | 不默认复用来源未独立确认的品牌纹理或美术 |

## 考虑过的替代方案

### 只观察效果，完全从零重写

可以降低代码归属管理成本，但会丢失 MIT 许可已经允许的合法复用价值，也难以准确说明实际参考关系。因此拒绝，选择透明记录来源和许可。

### 直接复制整个仓库

能最快获得近似画面，但会引入旧 API、工程链、DOM 假设、单路线玩法和不确定素材。这与“保留本项目 Core 和三端适配”的约束冲突，因此拒绝。

### 不记录参考来源

即使只改编一部分代码，MIT 仍要求在副本或实质部分中保留版权和许可声明。省略归属不符合许可也不利于后续评审，因此拒绝。

## 后果

- 团队可以在清晰边界内复用成熟动作与场景思路，同时保留本项目的玩法、确定性和三端架构。
- 仓库和发行包必须携带相关 MIT 归属。
- 引用与资产来源必须与代码评审一样可追溯；“只是参考”不能代替许可合规。
- 正式发布前仍需项目方做完整知识产权与平台审核；本 ADR 只记录开源复用决策，不构成法律意见。
