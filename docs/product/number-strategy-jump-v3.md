# 数值跳台 v3 产品基线

## 文档状态

已实现原型。本文保存 Arena V1 方向确认前的产品定义，不代表下一阶段目标。

## Users

面向使用手机竖屏游玩的轻度策略玩家。玩家需要在短局中同时判断左右数值运算路线与真实蓄力落点，用单手触控完成连续跳跃。

## Product Purpose

把“有限步数内精确命中目标值”的数值策略与可感知、可失败的物理跳跃结合起来。成功意味着玩家既选择了正确运算路径，也通过蓄力让角色真实落在目标平台上；表现层不能替玩家修改数值、碰撞或偏心落点。

## Brand Personality

克制、清晰、灵巧。画面应像柔和浅灰影棚中的极简几何玩具，动作短促而有触感，信息始终服务于下一次路线决策。

## Anti-references

- 不回到狐狸精灵、数据城市、霓虹远景、节拍轨或连击仪表盘。
- 不做自动吸附落点、自动替玩家选择路线或由动画决定碰撞结论。
- 不使用覆盖游戏主体的虚拟摇杆、贯穿整屏的厚重操作轨道、玻璃卡片或装饰性多色发光。
- 不让镜头切换、长动画或特效阻断连续决策。

## Design Principles

- 每次输入都必须明确对应左路或右路，不依赖玩家猜测整屏热区。
- 先呈现核心状态，再用动作和色彩反馈选择、蓄力、落地与失败。
- 世界推进保持空间连续，镜头只解释状态变化，不制造额外突变。
- 数值、世界和碰撞是唯一玩法真相，Three.js 表现始终单向消费快照。
- 所有触控在单手竖屏、安全区和不同屏幕比例下都应可靠。

## Accessibility & Inclusion

- 交互目标遵循至少约 56 CSS px 的手机触控尺寸，并避开系统手势安全区。
- 文字与按钮保持清晰对比，不只依赖颜色表达当前选择。
- 支持减少动态效果：缩短世界推进并取消非必要的长距离镜头追踪。
- 触摸取消、切后台和多指输入不得意外提交跳跃。
