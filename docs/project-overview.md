# 项目概览

## 项目是什么

“数域跃迁”是一个竖屏单机数值策略跳跃游戏。玩家在当前平台上选择屏幕左侧或右侧的运算平台，按住对应箭头蓄力，松开后沿解析抛物线跳跃；只有真实落点进入目标平台顶面，才执行绑定运算并扣除步数。

当前仓库同时产出：

- Web 静态站点。
- 微信小游戏 IIFE 构建。
- 抖音小游戏 IIFE 构建。

项目不包含账号、排行榜、云存档或后端服务。治理目标中的“1000 名玩家”指 1000 名玩家分别在本机运行同一发布版本，主要风险是客户端稳定性、设备兼容性、配置可解性和资源生命周期，而不是服务端并发。

## 当前已实现能力

### 玩法

- 当前值、目标值和有限步数。
- 加、减、乘、除候选运算。
- 版本化 `easy@1`、`normal@1`、`hard@1`；当前产品只开放 normal。
- 每轮保证至少存在一条受限步数内的精确解路径。
- 左右双分支连续世界。
- 成功、短跳、越过或落在平台外的不同碰撞结果。
- 成功落地保留真实偏心 `x/z` 坐标。

### 跳跃与世界

- 按住时长映射到跳跃射程。
- 解析抛物线轨迹和固定步长更新。
- 矩形平台顶面碰撞。
- 稳定平台 ID、当前/候选/历史关系和有界历史。
- 相同 seed 与输入时长可得到确定结果。

### 三维表现

- Three.js/WebGL2 单上屏 Canvas。
- 独立 World Scene 与 HUD Scene。
- 正交相机、光照、阴影、平台 Mesh、程序化角色。
- 蓄力形变、空翻、拖尾、粒子和失败表现。
- 相机和世界原点使用同一过渡进度平滑推进。
- 底部左右按钮根据相机投影映射屏幕左右候选。
- Scene/Character 已版本化注册，默认程序化内容进入生产 Renderer；10 角色 Manifest 容量、切换、回退和销毁已有测试。

### 平台

- Web、微信和抖音画布与 WebGL2 适配。
- 指针/触摸输入、帧循环、尺寸与安全区、前后台回调。
- 平台层已暴露声音、震动、存储和分享能力。
- GameEvent 已驱动程序化声音与震动；两者默认开启、可独立关闭并版本化本地保存。
- Web 已阻止 Canvas 长按菜单、文本选择和拖拽默认行为。

### 质量基础

- 当前为 124 项自动化测试（40 项 Node 兼容/集成测试和 84 项 TypeScript workspace 测试）。
- 三档难度各有 10,000 seed 的可解性验证，默认 normal 还有固定 seed 与完整获胜回放基线。
- Core、Runtime、Difficulty、Gameplay、Task、Jump Engine 的维护源码已经迁移为 strict TypeScript，旧 Core/Runtime JS 已删除。
- GameplayRegistry/TaskRegistry 已接入 GameSession，测试证明 5+5 静态容量及按 ID 选择实际定义。
- Renderer、Content、Feedback 已迁入 TypeScript workspaces，旧 `src/render3d` JS 已删除。
- `npm run lint`、`npm run typecheck` 和包依赖边界已成为累积门禁。
- `npm run build` 同时生成 Web、微信和抖音产物。
- 架构测试会检查 Core/Runtime 与平台 API 泄漏、小游戏 DOM 泄漏和 IIFE 可打包性。
- 已记录 Three.js 与参考项目的许可证归属。

## 当前没有完成的能力

- 只有一个正式玩法、一个任务类型和一个角色。
- 平台存储能力尚未形成版本化游戏存档。
- 没有回放、存档迁移、诊断导出或匿名遥测。
- 平台、入口、Node 兼容测试和构建脚本仍为 JavaScript；Renderer 虽为 `.ts`，仍含过渡宽松类型，第四批前尚未达到全面 strict TypeScript。
- workspace 测试虽为 `.ts`，当前过渡测试 tsconfig 尚未 strict；第四批必须严格化。
- 尚无覆盖率阈值和 CI 门禁。
- 微信/抖音 iOS 与 Android 真机验收仍缺少当前版本证据。
- WebGL 上下文监听和恢复后重同步入口已独立，但尚未证明所有 GPU 资源在真机完整重建。

## 目标扩展能力

第四批治理结束后，架构必须在不修改同一应用族主流程的前提下支持：

- 至少 5 个静态注册的玩法定义。
- 至少 5 个静态注册的任务定义。
- 至少 10 个角色 Manifest。
- 版本化难度、存档和确定性回放。
- 所有人工维护源码、测试和构建工具全面 TypeScript。

治理阶段只交付扩展机制和契约证明，不默认交付 5 个正式玩法、5 个正式任务和 10 个正式美术角色。

## 权威入口

- 本地开发：`npm run dev`
- 局域网开发：`npm run dev:lan`
- 自动化测试：`npm test`
- 三端构建：`npm run build`
- 当前完整门禁：`npm run check`
- 生产 Web 预览：`npm run preview:lan`
