# ADR-007：提供用户可选的高低画质档位

## 状态

Accepted

## 日期

2026-07-15

## 背景

同一 WebGL2 构建需要覆盖 iPhone 13 Pro Chrome、其他移动浏览器和小游戏宿主。仅依靠设备猜测无法稳定代表散热、系统版本和宿主限制，运行中自动来回切档又会造成分辨率和阴影跳变。

## 决策

1. 单 Canvas 内容菜单提供“高画质”和“低画质”第四个选择轴。
2. 高档：DPR 上限 2、阴影 1024、粒子容量 72、拖尾容量 18、缓存 24 MiB、动态纹理 12 MiB、60 FPS 目标。
3. 低档：DPR 上限 1.5、阴影 512、粒子容量 24、拖尾容量 10、缓存 10 MiB、动态纹理 6 MiB、30 FPS 目标。
4. 两档都保持相同 HUD、输入、玩法、任务、跳跃、世界和碰撞结果。
5. 切档由 Renderer Facade 投影为资源、Stage 和 Effect 小契约；Scene 不依赖性能策略模块。
6. 选择保存在独立本地 key `number-strategy.render-quality`，不改变 SaveEnvelope@4 和回放确定性。
7. 只在用户明确选择时切档，不在游戏过程中自动振荡。

## 后果

- 用户可主动用视觉完整度换取稳定帧率。
- 切档会重建表现层特效池并重新 resize，但不重建 Application 会话。
- 画质设置损坏或不存在时回退高档；存储失败不阻断游戏。
