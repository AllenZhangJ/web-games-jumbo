# Arena V1 MatchCore 批量压测结果

## 状态

阶段 5 Rule/Core/Map 本机无渲染门禁已通过，最近复测于 2026-07-17。本轮包含三件装备、风场、动态 surface、装备波、地图快照与回放成本。结果只证明权威逻辑成本与稳定性，不等价于 Renderer、AI 和目标真机整帧性能。

## 方法

- 使用正式 60Hz、3 条命、180 tick 准备、120 秒突然死亡、150 秒硬结束配置。
- 两名参赛者都只通过标准 `InputFrame` 提交追击、走位和上下文动作；装备使用不存在特权调用。
- 连续运行 1,000 局；每 tick 检查位置、速度、计时和状态有限性。
- 每局验证权威时限、结果、事件上限与最终 hash；抽取 5 局验证 checkpoint、事件序列和最终 hash 回放一致。
- 压测前后显式触发 GC，记录回收后的堆增量。

执行命令：

```bash
npm run arena:stress
```

测量环境：Node.js 20.19.5，macOS arm64。

## 结果

| 指标 | 结果 | 门禁 |
|---|---:|---:|
| 完整对局 | 1,000 / 1,000 | 必须全部结束 |
| 权威 tick | 909,066 | 无固定上限 |
| 平均每局 tick | 909.066（约 15.15 秒） | 仅为压力脚本行为，不作平衡结论 |
| 最长对局 tick | 1,307 | 不超过权威时限 |
| 平均逻辑成本 | 0.069353 ms/tick | ≤ 0.25 ms/tick |
| 非有限状态 | 0 | 必须为 0 |
| 回收后堆增长 | 3,015,488 B | ≤ 33,554,432 B |
| 抽样严格回放 | 5 / 5 | 必须全部一致 |
| 唯一最终 hash | 1,000 | 用于发现意外状态复用 |
| 正常决胜 | 942 | 统计项 |
| 同时淘汰平局 | 58 | 统计项 |

总计产生 50,678 个权威事件，其中包含 3,000 次初始装备刷新、1,494 次拾取、1,476 次掉落、4,922 次淘汰和 3,864 次重生。普通压力输入大多在首个风场启动前结束，因此完整地图时间轴由独立的 [Stage 5 地图压测](arena-map-stress-results.md) 覆盖。

## 结论与边界

MatchCore 在加入地图权威后仍显著低于 `0.25ms/tick` 阶段预算，未发现无法结束、非有限状态、回放分叉、非法 surface 支撑、装备滞留或持续堆增长。渲染、AI 和目标真机接入后仍必须再次测量。
