# 项目概览

## 项目是什么

“数域跃迁”是一个竖屏单机数值策略跳跃游戏。玩家选择屏幕左侧或右侧的运算平台，按住对应箭头蓄力，松开后沿解析抛物线跳跃；只有真实落点进入目标平台顶面，才执行绑定运算并扣除步数。

仓库同时产出 Web 静态站点、微信小游戏 IIFE 和抖音小游戏 IIFE。项目不包含账号、排行榜、云存档或后端服务。“1000 名玩家”指 1000 名玩家分别在本机运行同一发布版本，重点是客户端稳定性、设备兼容、配置可解和资源生命周期，而不是服务端并发。

## 当前已实现能力

### 玩法、任务与难度

- 当前值、目标值、有限步数与加减乘除候选。
- `easy@1`、`normal@1`、`hard@1` 版本化难度；产品只开放 normal。
- 每轮存在受限步数内的精确解；三档各有 10,000 seed 自动验证。
- Gameplay/Task 注册表已进入真实 `GameSession` 与单 Canvas 内容选择流程。
- 已交付 5 个正式玩法和 5 个正式任务；任务在成功落地后决定完成与失败，玩法声明兼容任务。

### 跳跃、世界与三维表现

- 确定性 RNG、解析轨迹、固定步长、真实矩形顶面碰撞和偏心落点。
- 稳定平台 ID、左右候选、历史窗口与成功落地事务回滚。
- Three.js/WebGL2 单上屏 Canvas、World/HUD Scene、正交相机、光影、程序化角色、粒子和拖尾。
- 相机与世界原点共享过渡进度；按钮按投影映射屏幕左右候选。
- Scene/Character 版本化注册、回退与销毁；10 个正式程序化角色具有独立轮廓、附件、配色和动作风格。

### 平台、反馈与本地存档

- Web、微信和抖音平台包提供 Canvas、WebGL2、输入、帧、尺寸/安全区、前后台、音频、震动、存储与分享端口。
- Web 阻止 Canvas 长按菜单、文本选择、拖拽和相关默认手势。
- GameEvent 驱动原创程序化声音与震动；两者默认开启、可独立关闭并持久化。
- `SaveEnvelope@4` 保存 seed、难度/玩法/任务/角色身份与确定性动作；支持 v1/v2/v3 fixture 迁移。
- 首帧前恢复并重放；损坏或不兼容存档隔离后回到新会话；只提供本地存档和本地诊断，不上传玩家数据。

### 工程质量

- 模块化单体使用 private npm workspaces，核心、应用、内容、反馈、持久化、渲染与平台边界独立。
- 所有维护源码、测试、入口、平台适配和构建/审计工具均为 strict TypeScript；零旧 JS 自动门禁已启用。
- 完整门禁包含 lint、typecheck、全量测试、80/70 覆盖率、1000 会话与 100 局资源 soak、依赖/资产/许可证审计和三端构建。
- GitHub Actions、CODEOWNERS、CHANGELOG、发布清单与三端产物预算已进入仓库。
- Three.js 与参考项目 `web-jump` 的 MIT 归属和完整许可文本随三端产物发行。

## 当前没有完成或仍待验证

- 当前内容均开放选择，但尚未建设等级、成就、付费或进度解锁系统。
- 没有账号、排行榜、云同步、远程配置、在线插件或匿名遥测。
- Feedback 没有玩家可见设置页；只有可持久化 API。
- 微信/抖音 iOS/Android 真机验收仍缺当前版本证据。
- WebGL context 恢复后完整 GPU 资源重建与低端真机长时间性能仍需设备证明。
- 主分支保护需仓库管理员在 GitHub 设置中启用，仓库文件不能替代服务端规则。

## 权威入口

- 本地开发：`npm run dev`
- 局域网开发：`npm run dev:lan`
- 自动化测试：`npm test`
- soak：`npm run test:soak`
- 三端构建：`npm run build`
- 完整门禁：`npm run check`
- 生产 Web 预览：`npm run preview:lan`
