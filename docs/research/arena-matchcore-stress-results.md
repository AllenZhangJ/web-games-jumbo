# Arena V1 MatchCore 批量压测结果

## 状态

阶段 4 Rule/Core 本机无渲染门禁已通过，最近复测于 2026-07-17。本轮已包含大锤、锁链、盾牌的拾取、动作、冷却、掉落、快照和回放成本。结果只证明权威逻辑成本与稳定性，不等价于 Renderer、AI 和目标真机整帧性能。

## 方法

- 使用正式 60Hz、3 条命、180 tick 准备、120 秒突然死亡、150 秒硬结束配置。
- 两名参赛者都只通过标准 `InputFrame` 提交追击、走位和上下文动作；装备使用不存在特权调用。
- 连续运行 1,000 局；每 tick 检查位置、速度、计时和状态有限性。
- 每局验证权威时限、结果、事件上限与最终 hash；抽取 5 局验证 checkpoint、事件序列和最终 hash 回放一致。
- 压测前后显式触发 GC，记录回收后的堆增量。

执行命令：

```bash
npm run arena:stress
```

测量环境：Node.js 20.19.5，macOS arm64。

## 结果

| 指标 | 结果 | 门禁 |
|---|---:|---:|
| 完整对局 | 1,000 / 1,000 | 必须全部结束 |
| 权威 tick | 980,121 | 无固定上限 |
| 平均每局 tick | 980.121（约 16.34 秒） | 仅为压力脚本行为，不作平衡结论 |
| 最长对局 tick | 1,450 | 不超过权威时限 |
| 平均逻辑成本 | 0.026117 ms/tick | ≤ 0.25 ms/tick |
| 非有限状态 | 0 | 必须为 0 |
| 回收后堆增长 | 2,226,624 B | ≤ 33,554,432 B |
| 抽样严格回放 | 5 / 5 | 必须全部一致 |
| 唯一最终 hash | 1,000 | 用于发现意外状态复用 |
| 正常决胜 | 990 | 统计项 |
| 同时淘汰平局 | 10 | 统计项 |

总计产生 51,202 个权威事件，其中包含 3,000 次装备刷新、1,987 次拾取、1,907 次掉落、4,932 次淘汰和 3,922 次重生。超时和突然死亡分支由独立自动化边界测试覆盖；本压力输入倾向主动交战，因此没有触发超时。

## 结论与边界

MatchCore 在本机压力环境中仍显著低于 `0.25ms/tick` 的阶段预算，未发现无法结束、非有限状态、回放分叉或持续堆增长。地图机关、渲染和目标真机接入后仍必须再次测量，不能把本结果外推为目标手机的 60 FPS 结论。
