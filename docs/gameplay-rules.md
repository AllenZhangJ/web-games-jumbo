# 游戏规则与玩法

本文记录 v3 当前已经实现的规则。玩法真相位于 `packages/gameplay` 与 `packages/jump-engine`，`packages/application` 负责用例编排；`src/render3d` 只消费快照和事件，不能修改数值、世界坐标或碰撞结论。

## 1. 操作

- 游戏为竖屏；等待状态下只有按住底部 `↖` 按钮才能选择屏幕左侧候选，按住 `↗` 按钮才能选择屏幕右侧候选。候选会按当前相机投影动态映射，不依赖 Core 数组顺序。
- 按住时间直接累计为蓄力时间，松手后起跳；一次只跟踪一个触点。
- 点击角色、平台或场景空白不开始蓄力；跳跃、落地和镜头推进期间两个按钮不可用。
- Web 浏览器会阻止 Canvas 长按菜单、文本选择和拖拽默认行为，长按只用于游戏蓄力。
- 取消触摸会撤销本次蓄力，不扣步数。
- v3 不显示力度条。角色与当前平台会随蓄力同步压缩，所选候选变为青色并显示细轮廓。
- 顶部显示“当前、目标、剩余”，右上提供暂停和重开。
- 暂停冻结逻辑更新；进入后台自动暂停，返回前台后由玩家继续。

## 2. 连续世界

- 世界使用真实 `x/y/z` 坐标；`x/z` 为平台平面，`y` 为高度。
- 初始包含一个当前平台和左右两个候选平台。
- 候选前向距离为 `3.8～4.25`，侧向距离为 `1.25～1.65` 世界单位。
- 平台碰撞顶面半宽为 `1.05`、半深为 `0.75`；Three.js Mesh 的视觉厚度不改变碰撞体。
- 成功落地后：旧当前平台进入历史；被选候选保持 ID 并晋升当前；未选候选回收；再生成两个候选。
- 世界最多保留最近 3 个历史平台。
- 表现层通过平移 `worldRoot` 重新构图，但绝不修改核心绝对坐标。
- 成功落地后先保留约 `80 ms` 的落地确认，再让世界原点与镜头焦点共享约 `440 ms` 的 `ease-in-out cubic` 位移，避免两套缓动叠加造成切镜跳变。

## 3. 蓄力、轨迹与碰撞

蓄力到射程的映射：

```text
p = clamp((蓄力毫秒 - 80) / (1200 - 80), 0, 1)
射程 = 0.8 + (7.6 - 0.8) × p^1.18
```

- 水平轨迹沿“角色真实位置 → 所选平台中心”的方向线性前进。
- 垂直轨迹使用归一化抛物线。
- 跳跃时长随力度从 `520 ms` 增至 `820 ms`，最高高度从 `1.1` 增至 `2.2`。
- 逻辑固定以 60Hz 更新，碰撞结论不依赖屏幕刷新率。
- 最终脚底 `x/z` 位于目标平台矩形顶面内才算成功。
- 成功只把 `y` 设置为平台顶面，绝不把 `x/z` 吸附中心。
- 落点偏前或偏后会真实改变下一跳距离和需要的蓄力。
- 未到近边为 `short`，越过远边为 `overshoot`；失败不执行运算、不扣步数。

## 4. 数值策略

| 项目 | 规则 |
|---|---|
| 初始当前值 | 6～18 |
| 基础目标值 | 28～72 |
| 实际目标值 | `max(基础目标值, 初始值 + 12 + 回合数 × 2)` |
| 每局步数 | 7 |
| 允许结果范围 | -99～199 |

候选运算：

- `+2～+9`、`−2～−9`，只差 1 时可出现 `+1` 或 `−1`。
- 当前值大于 0 且不越界时可出现 `×2`。
- 当前值可整除时可出现 `÷2`、`÷3`，不产生小数。
- 两个候选中至少一个会缩短当前值与目标值的距离，左右顺序随机。
- 只有成功落地才执行平台绑定运算并扣除一步。
- 最后一步精确命中目标仍判胜；步数耗尽且未命中则失败。

v3 已删除节拍、连击、节拍轨、最高分和背景音乐系统。

## 5. Three.js 表现

- 使用 WebGL2、Three.js 正交相机、浅灰背景、阴影接收平面、一个半球光和一个方向光。
- 角色为红色球头与截锥身体；蓄力时压缩，起跳时回弹并完成一周空翻。
- 平台使用真实 Cube/Cylinder/Parcel Mesh；平台 ID 稳定映射到 Mesh。
- 候选平台上方显示代码生成的运算标签。
- 成功落地触发粒子；跳跃产生轻量拖尾。
- `short`、`overshoot` 等失败会触发不同方向的倾倒与坠落表现，但不改变核心失败结论。
- 暂停、胜利、失败通过同一个 WebGL Canvas 上的独立 HUD Scene 呈现。

## 6. 平台与构建

- Web、微信、抖音共用数值、世界、碰撞、运行时和 Three.js 表现代码。
- 平台差异只位于 `src/platform`：主 Canvas、WebGL2、离屏文字 Canvas、输入、帧循环、安全区、生命周期、震动、存储和分享。
- 微信和抖音构建均声明 `deviceOrientation: portrait`。
- Canvas 按 DPR（最高 2）和安全区缩放；桌面宽窗口保持竖屏游戏构图。
- 现代 Three.js 构建要求宿主提供真实 WebGL2；开发工具成功不能替代 iOS/Android 真机验证。
